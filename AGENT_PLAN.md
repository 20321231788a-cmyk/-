# 智能体参与计划

## 目标

把二手车直播投流工作拆成四个智能体：用户、策划、执行、验收。四者围绕同一套账号数据和门店经营指标协作，减少只看播放量、不看有效线索和到店成交的问题，同时让老板、投手、主播都能看到自己下一步该做什么。

## 1. 用户智能体

职责：

- 根据用户角色解释当前页面和数据含义。
- 在缺数据、导入失败、计划缺项时给出下一步。
- 把复杂投放术语转成可执行动作。

交付：

- 新手引导
- 缺数据提醒
- 下一步建议
- 风险提示
- 可复制给 Codex 的指令

验收口径：

- 不替用户确认高风险动作。
- 解释建议背后的业务原因。
- 优先给最短可执行路径。

## 2. 策划智能体

职责：

- 分析账号视频、直播、线索、到店和成交数据。
- 判断哪些视频适合 DOU+ 测试，哪些直播适合本地推放大。
- 制定下一周预算、地域、人群、投放目标和内容选题。

交付：

- 周投流计划
- 当日直播预热策略
- 视频复拍选题
- 加投和停投规则

验收口径：

- 计划必须包含预算、目标、素材、地域、人群、KPI。
- 计划必须说明为什么投、何时停、何时加。

## 3. 执行智能体

职责：

- 把策划结果拆成可操作任务。
- 生成 DOU+、巨量本地推、线索广告计划草稿。
- 生成直播间讲车脚本、评论回复和销售跟进话术。

交付：

- 投放计划草稿
- 开播前执行清单
- 直播间自动回复建议
- 车辆讲解脚本
- 销售跟进清单

约束：

- 不自动发布真实广告计划。
- 不做刷屏、诱导、虚假承诺和绕过平台审核的动作。
- 涉及价格、金融、车况承诺时必须人工确认。

## 4. 验收智能体

职责：

- 每天收播后复盘消耗和转化。
- 判断素材、直播、投放计划是否达标。
- 给出下一轮修正动作。

交付：

- 日复盘报告
- 素材评分
- 投放验收结论
- 下一轮预算调整建议

验收口径：

- 核心指标：有效线索、预约、到店、成交。
- 辅助指标：播放、互动、涨粉、主页访问、私信。
- 对归因缺口、异常数据和无效线索必须明确标记。

## 协作流程

1. 用户智能体判断当前角色和任务，提示最短操作路径。
2. 网页通过表格或 AI/Codex 数据包导入视频、直播、投放、线索和成交数据。
3. 策划智能体生成投放策略和内容计划。
4. 执行智能体生成广告草稿、直播脚本和跟进清单。
5. 人工确认后在官方平台执行投放。
6. 验收智能体根据结果复盘，输出下轮优化。

## 与 Codex / AI 编程工具连接

- CLI：通过 `npm run bridge -- sample|prompt|validate` 读写 `data/outbox` 和 `data/inbox`。
- MCP：后续封装 `get_account_snapshot`、`write_strategy_result`、`validate_strategy_result`、`create_campaign_draft` 等工具。
- 安全边界：AI 工具只生成草稿和建议，不直接创建广告计划，不自动回复评论。
